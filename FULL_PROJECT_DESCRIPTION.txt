============================================================
1) PROJECT PURPOSE
============================================================
AGF is a multi-role emergency roadside + fuel-delivery operations platform built on Next.js App Router.

Primary roles:
- Public visitor (Landing + auth pages)
- User (customer requesting service/fuel)
- Worker (roadside responder)
- Fuel Station (fuel supply + COD settings + earnings)
- Admin (operations + pricing + payouts + controls)

Primary business capabilities:
- Create, assign, track, and close service requests
- Fuel station assignment by proximity + stock + COD policy
- Online payments and COD eligibility enforcement
- Settlement accounting across platform, worker, and station
- Worker and fuel-station payout workflows
- Connectivity telemetry + heatmap + zone warnings

============================================================
2) TECH STACK / ARCHITECTURE SUMMARY
============================================================
Frontend:
- Next.js App Router (mixed TSX/JS)
- React hooks and client-side guards
- Global CSS theme + role-specific layout blocks
- Leaflet/react-leaflet maps + leaflet.heat overlays

Backend (inside same Next.js app):
- Route handlers under app/api/*
- Database adapter abstraction in database/db.js
- SQLite default runtime, optional MySQL/PostgreSQL support

Infra features implemented in code:
- Environment-based DB client detection
- SQL placeholder normalization for postgres
- JWT token generation middleware
- Razorpay order/verify/webhook and payout integrations
