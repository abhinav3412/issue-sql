======================================================================
1) SYSTEM OVERVIEW
======================================================================
AGF is a full-stack multi-role platform for:
- emergency roadside support
- fuel delivery assignment
- COD/online payments
- settlements, payouts, and operational controls
- connectivity telemetry and risk overlays

Role matrix:
- Public: landing + auth + policy pages
- User: request creation, payment, tracking, rating
- Worker: request execution, location updates, earnings, payout visibility
- Fuel Station: stock, COD settings, settlement visibility, bank details
- Admin: global controls, monitoring, payouts, pricing, COD governance

Operational style:
- Next.js App Router monorepo architecture
- App pages and APIs live in one codebase
- DB abstraction supports sqlite/mysql/postgres runtime modes

6) SMALL FEATURE/LOGIC DETAILS (MICRO-BEHAVIORS)
======================================================================

- Signup validates country-specific phone structure before DB insert.
- Duplicate email handling maps DB unique failures to 409 response.
- Fuel-station signup initializes both petrol and diesel stock rows.
- Assignment logic considers business policy, not only distance.
- COD path can be blocked by station trust/limit constraints.
- Map layer supports street/satellite toggle controls.
- Fuel station markers are hidden below zoom threshold to reduce clutter.
- Optional station double-click can open Google Maps directions.
- User connectivity warning shows weak/no-coverage prediction by distance.
- Connectivity report posting is throttled by movement + cooldown logic.
- Heatmap includes zoom-adaptive radius behavior.
- Dev-safety guard exists for Leaflet heat redraw race scenario.
- Recenter control returns map to active user/default location.
